# Commercial Batch Plan v0.5

## Purpose

This document defines the path from prototype to first commercial batch.

The project must avoid jumping directly from idea to mass production.

Hardware must be scaled carefully.

---

## Batch ladder

```text
1 unit       proof of concept
5 units      engineering prototype
20 units     internal and partner testing
50 units     developer kit batch
200 units    pilot commercial batch
1,000 units  first commercial production
5,000+       scale stage
```

---

## Why the ladder matters

Hardware mistakes become expensive when repeated hundreds or thousands of times.

The first batches must discover:

```text
audio quality issues
Wi‑Fi setup problems
case design weaknesses
battery/power problems
support burden
API cost per active node
customer confusion
manufacturing yield
```
