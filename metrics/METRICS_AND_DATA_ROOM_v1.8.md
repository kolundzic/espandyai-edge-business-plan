# Metrics and Data Room v1.8

## Purpose

This document defines the metrics and data room structure needed for investor and partner conversations.

---

## Key metrics to track

### Product metrics

```text
number of prototypes built
successful demo runs
device uptime
average response time
audio success rate
local log success rate
Director event success rate
```

### Business metrics

```text
waitlist signups
pilot inquiries
pilot conversion rate
dev kit preorders
average selling price
estimated gross margin
monthly recurring revenue
customer acquisition cost
```

### Usage metrics

```text
commands per node per day
successful answers
failed requests
approval-required actions
blocked actions
notes captured
documents queried
```

### Support metrics

```text
setup time
support tickets per device
common failure points
return/replacement rate
```

## Evidence checklist

```text
prototype photos
demo video
GitHub repo link
architecture diagrams
pilot outreach list
LOIs if available
waitlist export
financial model
BOM estimate
roadmap
legal disclaimer
```

---

## Investor-grade proof hierarchy

Weak proof:

```text
idea only
```

Better proof:

```text
documents + diagrams
```

Stronger proof:

```text
working demo
```

Very strong proof:

```text
pilot customer
```

Best early-stage proof:

```text
paid pilot + repeatable workflow + recurring software revenue signal
```
