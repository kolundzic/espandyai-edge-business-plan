# Legal and Positioning Notes

## 1. Independent Project Position

ESPAndyAI EDGE / Director Node is an independent AndyAI project.

It is not affiliated with, endorsed by, or presented as an official component of Espressif, ESP-Claw, OpenClaw, or any third-party project.

## 2. Naming Policy

Avoid naming repos or products in a way that implies ownership or official relationship with third-party projects.

Recommended names:

```text
andyai-mini-node-sd-canon
andyai-director-edge-node
espandyai-edge
```

Avoid names that may imply official association:

```text
esp-claw-andyai
official-esp-claw-node
openclaw-andyai-device
```

## 3. Reference Policy

Third-party frameworks may be cited as public references or market signals where appropriate.

Example wording:

```text
ESP-Claw is a public Espressif open-source project that validates the broader market direction toward agentic IoT and edge-device behavior. ESPAndyAI EDGE is an independent AndyAI architecture with its own Director, trust, and business workflow layers.
```

## 4. Open-Source Compliance

If any third-party open-source code is used, the project must:

```text
read the license
preserve license notices
include attribution
track dependencies
avoid copying branding
document modifications
```

## 5. Hardware Compliance

For commercial hardware sales, certification planning may be required.

Potential compliance areas:

```text
FCC
CE
radio/EMC testing
battery safety
country-specific import rules
consumer safety labeling
```

## 6. Data and Privacy

Because the product may process voice and business information, production versions should define:

```text
voice data handling
retention policy
local logs
user consent
business data boundaries
secure transport
access control
delete/export process
```

## 7. Safety

The system should clearly distinguish:

```text
safe local actions
actions requiring confirmation
blocked actions
admin-only actions
```

Human approval is not optional for high-risk business workflows.
