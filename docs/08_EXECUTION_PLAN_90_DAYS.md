# 90-Day Execution Plan  
## ESPAndyAI EDGE / Director Node

## Purpose

This document defines the first 90 days of execution for the ESPAndyAI EDGE / Director Node project.

The goal is not to build a perfect final product.  
The goal is to create a credible chain of proof:

```text
physical prototype -> demo video -> landing page -> pilot offer -> user feedback -> investor package
```

## Core Principle

```text
Do not sell a dream first.
Show a working node first.
```

The first proof must be simple, visual, and impossible to misunderstand.

## 90-Day Targets

By the end of 90 days, the project should have:

1. one working ESPAndyAI EDGE prototype
2. one public-facing demo video
3. one GitHub business/technical documentation repo
4. one landing page draft
5. one pilot offer
6. one investor memo
7. one 10-slide pitch deck script
8. at least 10-30 early interest contacts
9. at least 2-5 serious pilot conversations

## Month 1 — Proof of Life

### Goal

Build the first proof that the device can communicate with AndyAI Director.

### Required demo

```text
User presses a button.
User speaks a short command.
ESPAndyAI EDGE sends the event to Director.
Director produces a response.
The node speaks or displays the response.
The event is logged locally or in the dashboard.
```

### Technical tasks

- select one ESP32-S3 development board
- connect microphone
- connect speaker/amplifier
- connect optional OLED display
- add microSD logging
- establish Wi-Fi connection
- implement simple event packet
- implement Director test endpoint
- implement response playback or display

### Business tasks

- write one-page product explanation
- define 3 strongest use cases
- prepare early demo narrative
- document the prototype in photos and short clips

### Milestone

```text
ESPAndyAI EDGE Node breathes.
```

## Month 2 — Demonstrable Prototype

### Goal

Move from "wires on table" to a prototype that can be shown to partners.

### Technical tasks

- improve firmware stability
- add device ID
- add simple provisioning instructions
- add local log format
- create basic Director dashboard view
- add safe command list
- add at least 3 demo workflows

### Demo workflows

1. voice note capture
2. document/question event
3. local action such as LED/status signal

### Business tasks

- create 90-second demo script
- prepare pilot offer
- prepare landing page copy
- prepare investor one-pager
- define early pricing

### Milestone

```text
The node is understandable to a non-engineer.
```

## Month 3 — Market Signal

### Goal

Collect the first external validation.

### Technical tasks

- create 3-5 repeatable prototype units if possible
- document setup process
- prepare simple troubleshooting guide
- record final demo video
- publish/update GitHub repo

### Market tasks

- contact 20-50 potential pilot partners
- publish LinkedIn/YouTube demo clips
- collect feedback
- create waitlist form
- identify first paid pilot target

### Investor tasks

- prepare investor memo
- prepare pitch deck
- prepare market-risk narrative
- prepare budget ask

### Milestone

```text
The project has evidence, not only vision.
```

## 90-Day Success Criteria

Minimum success:

```text
1 working prototype
1 demo video
1 landing page
10 interested contacts
1 serious pilot discussion
```

Strong success:

```text
3-5 prototypes
30+ waitlist contacts
2-5 pilot discussions
1 LOI or paid pilot path
```

Excellent success:

```text
first paid pilot
developer kit interest
investor meeting scheduled
clear unit economics
```

## Key Decision After 90 Days

At the end of 90 days, decide between:

1. continue bootstrapped with pilot sales
2. raise a small pre-seed round
3. create first 20-50 dev kit batch
4. narrow to one vertical market first

## Recommended First Vertical

The strongest first vertical is:

```text
AI documentation and office workflow node
```

Reason:

- easier than medical/elderly-care
- clearer business value
- low physical risk
- strong connection to AndyAI Director and knowledge systems
- easier demo for investors and business partners
