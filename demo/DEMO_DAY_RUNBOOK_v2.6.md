# Demo Day Runbook v2.6

## Purpose

This file defines how to run a live demo for partners, pilot clients, or investors.

---

## Pre-demo checklist

```text
device charged/powered
Wi‑Fi ready
Director endpoint online
dashboard open
SD card inserted
speaker tested
button tested
backup demo video ready
printed one-pager ready
pitch deck ready
```

---

## Demo order

```text
1. Explain the problem in one sentence.
2. Show the device.
3. Explain Director as the brain.
4. Press button.
5. Speak command.
6. Show Director receives event.
7. Device replies.
8. Show dashboard/audit log.
9. Show local log concept.
10. Explain pilot workflow.
```

---

## Demo command examples

```text
“AndyAI, save this as a pilot note.”
“AndyAI, what is the current node status?”
“AndyAI, record this workshop measurement.”
“AndyAI, ask the documentation system what the next step is.”
```

---

## If live demo fails

Do not panic.

Use backup flow:

```text
show architecture
show recorded demo video
explain prototype status honestly
show logs/screenshots
return to business value
```

---

## Demo success criteria

The audience understands:

```text
device is physical interface
Director is intelligence layer
human approval is core
workflow value is practical
pilot can start small
```
