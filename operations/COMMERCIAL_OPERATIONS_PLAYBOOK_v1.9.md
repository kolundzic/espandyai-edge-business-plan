# Commercial Operations Playbook v1.9

## Purpose

This playbook defines how to operate the business once pilots and early customers begin.

---

## Operating cycle

```text
lead
discovery
workflow selection
proposal
pilot setup
usage period
pilot report
commercial offer
renewal/expansion
```

---

## Lead qualification

Good lead:

```text
has a clear workflow problem
has physical workspace
can name users
has documents or procedures
accepts pilot scope
has budget
```

Weak lead:

```text
wants vague AI magic
cannot define workflow
has no decision maker
expects custom hardware for free
```

---

## Proposal structure

```text
problem
pilot workflow
devices included
Director configuration
timeline
success metrics
price
support terms
next step
```

---

## Support process

```text
setup guide
onboarding call
known issues page
support email
pilot check-in
pilot report
```

---

## Internal weekly review

Each week review:

```text
new leads
active pilots
device issues
support tickets
usage metrics
costs
content needed
investor updates
```

---

## Expansion logic

Do not expand because the idea sounds exciting.

Expand when data shows:

```text
people use the node
workflow is repeated
customer sees value
support burden is manageable
Director subscription is justified
```
