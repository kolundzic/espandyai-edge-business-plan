# Post-Launch Feedback Loop v2.8

## Feedback sources

```text
landing page form
pilot discovery calls
developer waitlist
investor questions
demo reactions
GitHub issues
support emails
```

---

## Weekly synthesis

Every week, summarize:

```text
top 5 questions
top 5 objections
top 5 requested features
top 3 confusing points
top 3 strongest reactions
```

---

## Decision rules

If many people ask the same question:

```text
update FAQ
```

If many people misunderstand the product:

```text
update landing page and pitch
```

If a feature is requested by multiple pilots:

```text
add to roadmap
```

If support burden is high:

```text
simplify setup
```

---

## Learning loop

```text
market signal
→ documentation update
→ product update
→ pilot update
→ investor narrative update
```

This is the commercial version of TAPFORGE learning.
